# 📤 Distribution Agents - 배포팀

> publish-agent → analytics-agent → feedback-agent
> GATE 4 승인 후 순차 실행

---

## 1. Publish Agent (배포)

### 역할
완성된 영상을 전 플랫폼에 동시 업로드한다.

### 트리거
- GATE 4 승인 후 자동 실행

### 업로드 플랫폼

| 플랫폼 | 우선순위 | 도구 |
|--------|----------|------|
| Instagram Reels | ✅ 필수 | Upload-Post.com API |
| X (Twitter) | ✅ 필수 | Upload-Post.com API |
| YouTube Shorts | 선택 | Upload-Post.com API |
| TikTok | 선택 | Upload-Post.com API |
| Threads | 선택 | Upload-Post.com API |

### 업로드 정보
각 플랫폼별로 설정:
```
- 제목/캡션
- 해시태그 (플랫폼별 최적화)
- 썸네일 (자동 생성 또는 지정)
- 업로드 시간 (최적 시간대 자동 계산)
- 언어/자막 설정
```

### 실행 흐름
```
1. 플랫폼별 메타데이터 생성 (제목, 해시태그 등)
2. Upload-Post.com API 호출 (1회로 전 플랫폼 동시 업로드)
3. 업로드 결과 확인
4. 구글 시트에 업로드 정보 기록
5. analytics-agent 타이머 시작
```

### 사용 도구
- `Upload-Post.com API`: 멀티플랫폼 업로드
- `Claude API`: 플랫폼별 캡션/해시태그 생성
- `구글 시트`: 업로드 기록

### 출력
- 업로드 완료 확인
- 각 플랫폼 게시물 URL

---

## 2. Analytics Agent (반응 수집)

### 역할
업로드된 콘텐츠의 성과 지표를 주기적으로 수집한다.

### 트리거
- 업로드 후 1h / 6h / 24h / 72h 자동 실행

### 수집 지표

| 지표 | Instagram | X |
|------|-----------|---|
| 조회수 | ✅ | ✅ |
| 시청완료율 | ✅ | - |
| 평균시청시간 | ✅ | - |
| 이탈구간 | ✅ | - |
| 좋아요 | ✅ | ✅ |
| 저장수 | ✅ | ✅ (북마크) |
| 공유수 | ✅ | ✅ (RT) |
| 댓글수 | ✅ | ✅ (리플) |
| 댓글 감정분석 | ✅ | ✅ |

### 수집 주기별 목적
| 시점 | 목적 |
|------|------|
| 1h | 초기 반응 확인 (알고리즘 픽 여부) |
| 6h | 확산 추이 (바이럴 가능성) |
| 24h | 일일 성과 (안정적 지표) |
| 72h | 최종 성과 → GATE 5 알림 |

### 사용 도구
- `Instagram Graph API`: 인스타 지표 수집
- `X API v2`: X 지표 수집
- `Claude API`: 댓글 감정 분석
- `구글 시트`: 성과 데이터 저장

### 출력
- 성과 데이터 → 구글 시트 기록
- 72h 수집 완료 시 → **⛔ GATE 5** (수환 확인)

---

## 3. Feedback Agent (성과분석/피드백)

### 역할
72h 성과 데이터를 분석하고 다음 기획에 반영할 인사이트를 도출한다.

### 트리거
- GATE 5 승인 후 자동 실행

### 분석 항목

#### 개별 영상 분석
```
- 총점 산출 (100점 만점)
- 성공 요인 분석 (무엇이 먹혔는가)
- 실패 요인 분석 (무엇이 아쉬웠는가)
- 최적 업로드 시간 vs 실제 시간
- 댓글 주요 키워드/반응
```

#### 패턴 분석 (누적)
```
- 스타일별 평균 성과 비교
- 시간대별 성과 비교
- 키워드/주제별 성과 비교
- 플랫폼별 성과 차이
- 트렌드 변형 vs 오리지널 성과 비교
```

#### 다음 기획 반영사항
```
- 잘된 점 → 강화할 요소
- 아쉬운 점 → 개선할 요소
- 새로 발견한 패턴 → 실험 요소
- 청중 반응 키워드 → 다음 주제 힌트
```

### 사용 도구
- `Claude API`: 성과 분석/인사이트 생성
- `구글 시트`: 성과 DB 읽기/쓰기

### 출력
- 성과 분석 리포트
- 다음 기획 반영사항 → 구글 시트 기록
- **trend-agent로 피드백 전달 (선순환 완성)**

---

## 선순환 구조

```
feedback-agent 출력
    ↓
구글 시트 성과 DB 누적
    ↓
trend-agent (다음 날 06:00)
    ↓
idea-agent가 성과 DB 참조하여 아이디어 생성
    ↓
... (파이프라인 반복)
```

이렇게 데이터가 쌓일수록 아이디어 품질이 올라가는 선순환 구조.
